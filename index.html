TDD vs Testing

Test Driven Development

Definition

  1. write a "single" unit test describing an app's aspect
  2. run the test, which should fail because the app lacks that feature
  3. write "just enough" code, the simplest possible, to make the test pass
  4. "refactor" the code until it conforms to the simplicity criteria
  5. repeat, "accumulating" unit tests over time
  1. Small test.
  2. Get red flag.
  3. Get green flag.
  4. Refactor.
  5. Repeat.

Laws

You can't write any production code until you have first written a failing unit test.

You can't write more of a unit test than is sufficient to fail, and not compiling is failing.

You can't write more production code than is sufficient to pass the currently failing unit test.

Shrink debugging time. Courage(Сміливість) to make changes. Documentation. Design. [Testable]
